# 记忆翻牌游戏改进设计

## 概述

对现有记忆翻牌游戏进行全面改进，保持简约精致的视觉风格，增加高级动效、游戏功能和音效反馈。

## 设计决策

- **实现方案**: 渐进式改进（在现有代码基础上逐步添加功能）
- **视觉风格**: 简约精致（保持现有风格，增加动画和细节）
- **动画效果**: 高级动效（基础动画 + 粒子特效、连击效果）
- **音效**: 基础音效（翻牌、匹配、胜利等关键操作）
- **难度**: 保持现有3种难度（简单4×3、普通4×4、困难6×4）

---

## 第一部分：视觉改进

### 1. 翻牌动画

使用 CSS 3D 变换实现真实的翻转效果：
- 卡片以 Y 轴为中心翻转 180°
- 背面显示渐变紫色 + 问号
- 正面显示白色背景 + Emoji 图案
- 翻转时间 0.3s，使用 ease-in-out 缓动

### 2. 匹配成功特效

配对成功时的视觉反馈：
- 卡片背景变为浅绿色 (#d1fae5)
- 边框高亮显示 (#10b981)
- 轻微放大弹跳动画
- 粒子从卡片位置向四周飘散

### 3. 匹配失败反馈

配对失败时的视觉反馈：
- 卡片短暂抖动效果
- 0.6s 后翻回背面

### 4. 游戏胜利庆祝

完成所有配对后的庆祝效果：
- 全屏彩色纸屑掉落动画
- 成绩展示动画（步数、用时）
- 最佳成绩高亮显示

---

## 第二部分：功能设计

### 1. 提示功能

帮助玩家记忆卡牌位置：
- 页面新增"💡 提示"按钮
- 每局游戏限制 3 次使用机会
- 点击后所有卡牌短暂翻开 1 秒
- 按钮显示剩余次数：`💡 提示 (3)`

### 2. 排行榜系统

记录历史最佳成绩：
- 记录每个难度的最佳成绩（步数 + 用时）
- 存储在 localStorage 中
- 游戏胜利时显示当前成绩与最佳成绩对比
- 可查看各难度的历史最佳

### 3. 主题切换

多种卡牌主题可选：
- 🍎 水果（默认）：🍎🍊🍋🍇🍓🍒🥝🍑🥕🌽🍕🎂⚽🎸🚀🌈🐶🐱🦊🐼🐸🦋🌸⭐
- 🐶 动物：🐶🐱🦊🐼🐸🦋🦁🐯🐻🐨🐵🐔🐧🐴🦄🐙🦋🐝🐞
- 😀 表情：😀😁😂🤣😃😄😅😆😉😊😋😎😍😘🥰😗😙🥲😚
- ⚽ 运动：⚽🏀🏈⚾🎾🏐🏉🎱🏓🏸🥊🥋⛳🏆🥇🥈🥉
- 主题选择器位于难度选择旁边
- 切换主题会重新开始游戏

## 第三部分：音效设计

使用 Web Audio API 生成简单音效，无需加载外部文件。

### 音效列表

| 触发时机 | 音效描述 | 实现方式 |
|---------|---------|---------|
| 翻牌 | 短促清脆的"叮"声 | 高频正弦波 + 快速衰减 |
| 匹配成功 | 上升音调的"叮叮"声 | 两个连续上升的正弦波 |
| 匹配失败 | 低沉的"嗡"声 | 低频锯齿波 + 缓慢衰减 |
| 游戏胜利 | 欢快的旋律音序 | C-E-G-C 上行琶音 |

### 静音控制

- 页面右上角添加静音按钮 🔊/🔇
- 静音状态保存在 localStorage 中
- 默认开启音效

## 技术实现

### 代码组织

采用渐进式改进，在现有单文件基础上添加：

1. **CSS 部分**
   - 新增动画关键帧（翻转、弹跳、抖动、纸屑）
   - 新增粒子效果样式
   - 新增主题选择器样式

2. **JavaScript 部分**
   - 音效系统模块（AudioContext 管理）
   - 提示功能模块
   - 排行榜模块（localStorage 读写）
   - 主题管理模块
   - 粒子效果生成器

### 存储结构

```javascript
// localStorage 键值
'memory_best' -> {
  easy: { moves: 12, seconds: 25, date: timestamp },
  medium: { moves: 18, seconds: 42, date: timestamp },
  hard: { moves: null, seconds: null, date: null }
}

'memory_theme' -> 'fruits' | 'animals' | 'emojis' | 'sports'

'memory_muted' -> true | false
```
